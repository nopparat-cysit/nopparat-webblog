# Notifications + Supabase Realtime

## พฤติกรรม (Backend → ตาราง `notifications`)

| เหตุการณ์ | ใครได้รับ | เงื่อนไข |
|-----------|------------|----------|
| สร้างบทความใหม่ **หรือ** อัปเดตจาก Draft → **Published** (`status_id = 2`) | สมาชิกทุกคน (`users.role = 'user'`) | ไม่ส่งให้ `author_id` ของบทความ |
| สมาชิก (`role = 'user'`) **กดถูกใจ** บทความ | เจ้าของบทความ (`posts.author_id`) | ไม่แจ้งถ้าเป็นคนกดเอง |
| สมาชิก **คอมเมนต์** | เจ้าของบทความ | ไม่แจ้งถ้าคอมเมนต์เอง / ไม่แจ้งถ้าผู้คอมเมนต์ไม่ใช่ `role = 'user'` |

Express ใช้ **Postgres pool** (เชื่อม Supabase DB) `INSERT` ลง `notifications` — Realtime จะ push ไป client ที่ subscribe แล้ว

## ฝั่ง Frontend

1. ตั้ง `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY` ใน `.env`
2. หลัง **login** สำเร็จ: `refresh_token` จาก API + `supabase.auth.setSession` (ดู `LoginPage.jsx`, `AuthContext.jsx`)
3. **`useSupabaseNotifications`**: โหลดรายการ + subscribe `postgres_changes` (INSERT/UPDATE) ที่ `user_id` ตรงกับผู้ใช้
4. **`NavBar`**: แสดง dropdown (desktop) / แผง (mobile) จากข้อมูลจริง

## ตารางและ Realtime

```sql
create table if not exists public.notifications (
  id bigint generated by default as identity primary key,
  user_id uuid not null,
  type text,
  message text,
  meta jsonb default '{}'::jsonb,
  is_read boolean default false,
  created_at timestamptz default now()
);

alter publication supabase_realtime add table notifications;
```

## RLS (แนะนำ)

```sql
alter table public.notifications enable row level security;

create policy "notifications_select_own"
  on public.notifications for select
  to authenticated
  using (auth.uid() = user_id);

create policy "notifications_update_own"
  on public.notifications for update
  to authenticated
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
```

INSERT จาก Express ใช้ connection แบบ service/postgres จึงไม่โดน RLS

## หมายเหตุ

- ล็อกอินผ่าน `/login` (API) ต้องได้ `refresh_token` ใน response (`auth.mjs` ส่ง `refresh_token` แล้ว)
- แอดมินล็อกอินผ่านหน้าแยกที่ไม่ได้ `setSession` อาจไม่เห็น Realtime จนกว่าจะใช้ flow เดียวกับ member
