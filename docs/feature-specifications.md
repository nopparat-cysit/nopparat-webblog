## 1. Article detail page with comments, shares, and likes

**As a user, I can see the article detail page with comments, shares, and likes.**

### Description

This feature allows users to view an article detail page where they can see the full content of the article, along with comments, shares, and likes. Registered users and admins can interact by adding comments, while non-members can view the content but cannot interact.

### Scope of Work

- Develop a page that displays the full content of the article, including title, body, images, and author's info.
- Implement a section to show comments related to the article.
- Include display elements for sharing options and the number of likes.
- Design the article detail page to be visually appealing and responsive on both desktop and mobile devices.
- Ensure that all elements (article content, comments, shares, likes) are displayed clearly and accurately.

### User Acceptance Criteria

- **Criterion 1:** Users can view the full content of the article, including title, body, and images.
- **Criterion 2:** Users can view comments associated with the article.
- **Criterion 3:** Users can see the sharing options and the number of likes for the article.
- **Criterion 4:** The article detail page is visually appealing and functions correctly on both desktop and mobile devices.
- **Criterion 5:** All displayed information is accurate and up-to-date.

## 2. Member/admin like, share, and comment on articles

**As a member/admin, I should be able to like, share, and comment on the article.**

### Description

This feature allows members and admins to interact with articles by liking, disliking, sharing, and commenting. These interactions enhance engagement and feedback on articles, enabling users to express their opinions and share content.

### Scope of Work

- Implement functionality to allow members and admins to like and dislike articles, with visible counts for each.
- Develop a feature that enables members and admins to share articles on social media or other platforms.
- Create a comments section where members and admins can add their thoughts or feedback on the article.
- Ensure all interaction features (like, dislike, share, comment) are accessible and functional for both members and admins.
- Design the interaction components to be user-friendly and responsive on both desktop and mobile devices.
- Ensure that all interactions are updated in real-time and accurately reflected on the article page.

### User Acceptance Criteria

- **Criterion 1:** Members and admins can like and dislike articles, and see the updated like and dislike counts.
- **Criterion 2:** Members and admins can share articles using available sharing options.
- **Criterion 3:** Members and admins can add comments to articles, which are displayed in the comments section.
- **Criterion 4:** Interaction components are visible, functional, and responsive on both desktop and mobile devices.
- **Criterion 5:** All interactions (likes, dislikes, shares, comments) are updated in real-time and accurately reflect user actions.

## 3. Member/admin edit profile info

**As a member/admin, I should be able to edit my profile info (pic, username, reset password, etc.).**

### Description

This feature allows members and admins to edit their profile information, including profile picture, username, and other personal details. This functionality enables users to keep their profiles up-to-date and reflective of their current information.

### Scope of Work

- Develop an interface where members and admins can view and edit their profile information, including profile picture, username, and other personal details.
- Implement backend functionality to securely update and store changes to profile information.
- Ensure the profile editing interface allows for uploading profile pictures.
- Provide validation for profile fields to ensure data integrity (e.g., valid username format, appropriate image size).
- Design the profile editing interface to be user-friendly and responsive on both desktop and mobile devices.
- Include confirmation messages or notifications upon successful update of profile information.

### User Acceptance Criteria

- **Criterion 1:** Members and admins can access and use an interface to view and edit their profile information.
- **Criterion 2:** Users can successfully update their profile picture, username, reset password, and other personal details.
- **Criterion 3:** The system validates input fields to ensure proper format and size, providing appropriate error messages for invalid entries.
- **Criterion 4:** Users receive confirmation messages or notifications upon successful update of their profile information.
- **Criterion 5:** The profile editing interface is responsive and functions correctly on both desktop and mobile devices.
